# Agent Instructions

A minimal Astro 6 template ready for deployment.

## Overview

### Tech Stack

- **Astro 6** with Vercel Adapter (Default `output: 'static'` with on-demand server rendering for APIs)
- **Tailwind CSS 4** for styling
- **TypeScript** for type safety
- **Zod** for validation (especially for Content Collections)
- **Resend** for server-side email sending

### Data Source

> **⚠️ IMPORTANT: The base data source for this template is Markdown.**

By default, this project relies on **Markdown (.md) or MDX files** organized via **Astro Content Collections** for its data source. All projects start with Markdown, and content should be structured inside the `src/content/` directory with strictly typed schemas using Zod. However, the data source could be replaced by a Headless CMS in the future.

### Package Manager

> **⚠️ IMPORTANT: Always use `npm` for all package operations.**
>
> This project uses **npm** exclusively. Do NOT use `pnpm` or `yarn`.

## Development Guidelines

### Project Structure

All application code lives inside `src/`. Notable directories include:

- **`src/pages/`** — File-based routing (Astro pages and API endpoints)
- **`src/content/`** — Markdown/MDX files and collection configuration (`config.ts`)
- **`src/components/`** — Reusable UI components (Astro and React only)
- **`src/layouts/`** — Shared page layouts
- **`src/actions/`** — Server actions (if using Astro Actions for form endpoints)

### Resources Folder

The `resources/` folder at the project root contains client-provided materials:

- **Specifications** — Project briefs, requirements, client requests
- **Branding** — Brand guidelines, color palettes, typography
- **Assets** — Logos, images, icons, and visuals

Always review this folder first to understand the project scope and available assets. These files are **read-only reference materials** — do not modify them.

### Naming Conventions

- **Folders/directories**: `kebab-case` (e.g., `user-profile/`, `contact-form/`)
- **Components**: `PascalCase` (e.g., `UserProfile.astro`, `ContactForm.tsx`)

### Code Style

Format code according to `.prettierrc` and `.editorconfig`. Run `npm run format` to auto-format.

## Component Patterns

### Astro vs Client Components

> **⚠️ IMPORTANT: Prefer Astro Components (`.astro`). Push UI Framework components to the leaves.**

- By default, use **Astro components** which render entirely to static HTML (zero client-side JS)
- **React Only for UI Frameworks:** When interactivity is absolutely necessary, strictly use **React** components. Do not use Svelte, Vue, Solid, or any other UI framework. Include client directives (like `client:load`, `client:idle`, or `client:visible`)
- Isolate client logic into small, leaf-level components
- Never use client directives on layout wrappers unless required for global state providers

### Page Content Structure

Always separate data from presentation. Since this is a Markdown-first project:

1. **Content layer** (`src/content/`) — Markdown files containing frontmatter and body.
2. **Data Fetching** (Astro frontmatter) — Query content collections using `getCollection()` or `getEntry()`.
3. **Presentation layer** (Astro template) — HTML and components that consume the data.

```astro
---
// DATA
import { getCollection } from 'astro:content'
const posts = await getCollection('blog')
---

<!-- PRESENTATION -->
<ul>
  {
    posts.map((post) => (
      <li>
        <a href={`/blog/${post.slug}`}>{post.data.title}</a>
      </li>
    ))
  }
</ul>
```

## UI & Styling

### Responsive Design

Use a mobile-first approach — all pages must be fully responsive:

- Start with mobile layout, then enhance for larger screens using Tailwind's `sm:`, `md:`, `lg:` breakpoints
- Always include a mobile navigation menu with a hamburger toggle button (this may require a small interactive client component or plain JS in styling)
- Test layouts at all breakpoints: mobile (default), tablet (`md:`), desktop (`lg:`)

### Tailwind Styling

Use Tailwind classes directly in your templates. Key patterns: `flex`, `grid`, `p-4`, `m-2`, `gap-4`, `text-xl`, `font-bold`, `bg-*`, `text-*`, `rounded-lg`, `shadow-md`, `hover:*`, `dark:*`.

### Images

Place standard images in `public/` or `src/assets/` and use Astro's `<Image />` component for automatic optimization:

```astro
---
import { Image } from 'astro:assets'
import myImage from '../assets/photo.jpg'
---

<Image src={myImage} alt="Description" />
```

## Features

### API Endpoints & Server-Side Execution

While the site is statically generated by default (using `output: 'static'`), the **Vercel Adapter** allows specific components and endpoints to run server-side code when needed (like submitting forms or sending emails):

- Create API endpoints in `src/pages/api/` (e.g., `src/pages/api/submit.ts`) or use **Astro Actions** for safer, type-checked form handling. To ensure an endpoint runs dynamically, include `export const prerender = false`.
- These endpoints run securely on the server, meaning sensitive keys (like your Resend API key) are never exposed to the client.

### Email (Resend)

This project uses [Resend](https://resend.com) for sending emails securely from the server side. Skills are installed at:

- `.agents/skills/resend/` (generic folder for multiple standard coding agents)
- `.claude/skills/resend/` (symlink to the generic agents folder, specifically for Claude)

**Setup:** Set `RESEND_API_KEY` environment variable in `.env` (see `.env.example`).

**Usage:** Import Resend and trigger emails securely from within Astro endpoints or Astro Actions. Read `.agents/skills/resend/SKILL.md` for routing to sub-skills.

### SEO

This project systematically implements Search Engine Optimization (SEO). Skills are installed at:

- `.agents/skills/seo/` (generic folder for multiple standard coding agents)
- `.claude/skills/seo/` (symlink to the generic agents folder, specifically for Claude)

**Usage:** Always refer to the SEO skill when adding new pages or updating site structure. It covers metadata injection, sitemaps, `robots.txt` generation, JSON-LD structured data, and Open Graph / Twitter Card tags. Read `.agents/skills/seo/SKILL.md` for full implementation details.
